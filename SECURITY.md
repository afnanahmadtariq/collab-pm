# Security Policy - Collab-PM
# =============================================================================

## Supported Versions

| Version | Supported          |
| ------- | ------------------ |
| 1.x.x   | :white_check_mark: |
| < 1.0   | :x:                |

## Reporting a Vulnerability

We take security seriously. If you discover a security vulnerability, please
follow these steps:

1. **Do NOT** create a public GitHub issue
2. Email security concerns to: [your-email@example.com]
3. Include as much detail as possible:
   - Description of the vulnerability
   - Steps to reproduce
   - Potential impact
   - Suggested fix (if any)

### What to Expect

- **Acknowledgment**: Within 48 hours
- **Initial Assessment**: Within 1 week
- **Resolution Timeline**: Varies based on severity

### Severity Levels

| Level | Description | Response Time |
|-------|-------------|---------------|
| Critical | System compromise, data breach | < 24 hours |
| High | Significant security impact | < 72 hours |
| Medium | Limited security impact | < 1 week |
| Low | Minimal security impact | < 2 weeks |

## Security Best Practices

When contributing to this project:

1. Never commit secrets or credentials
2. Keep dependencies updated
3. Follow secure coding practices
4. Use environment variables for sensitive data
5. Implement proper input validation

## Disclosure Policy

We follow responsible disclosure:
- We will acknowledge your report promptly
- We will keep you informed of our progress
- We will credit you (if desired) when the issue is fixed
- We ask that you give us reasonable time to address the issue

Thank you for helping keep Collab-PM secure!
